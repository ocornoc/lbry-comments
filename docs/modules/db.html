Module db

A wrapper around SQLite3 providing the high-level interface of the backend.

db.claims

new (claim_uri)
Inserts a new claim into the claims database. If the claim is already in the database, it doesn't error.

Parameters:

  • claim_uri string The permanent LBRY URI of the claim.

Returns:

    int 1 if it was added, 0 if it was already preset.

Or

  1. nil On error.
  2. string The error message.

Usage:

    db.claims.new("lbry://cool_example")
get_data (claim_uri[, int_ind=false])
Returns the data for a given claim. If int_ind == true, then the data is an array rather than having alphanumeric keys.

Parameters:

  • claim_uri string The permanent LBRY URI of the claim.
  • int_ind bool (default false)

Returns:

    table The data of the given claim.

Or

  1. nil On error.
  2. string The error message.

Usage:

get_comments (claim_uri[, int_ind=false])
Returns the TLCs for a claim. A "TLC" is a "top-level comment", IE a comment that isn't a reply. If int_ind == true, then each comment's data is an array rather than having alphanumeric keys.

Parameters:

  • claim_uri string The permanent LBRY URI of the claim.
  • int_ind bool (default false)

Returns:

    table An array of comments' data.

Or

  1. nil On error.
  2. string The error message.

db.comments

new (claim_uri, poster, message)
Inserts a new TLC into the comments database. A "TLC" is a "top-level comment", IE a comment that isn't a reply.

Parameters:

  • claim_uri string The permanent LBRY URI of the claim to comment on.
  • poster string The name of the poster.
  • message string The message of the comment.

Returns:

    int The comment ID now associated with this comment.

Or

  1. nil On error.
  2. string The error message.

Usage:

    db.comments.new("lbry://cool_example", "my_username", "nice vid!")
new_reply (parent_id, poster, claim_uri)
Inserts a new reply into the comments database.

Parameters:

  • parent_id int The ID of the comment that this is a reply to.
  • poster string The name of the poster.
  • claim_uri string The message of the comment.

Returns:

    int The comment ID now associated with this comment.

Or

  1. nil On error.
  2. string The error message.

Usage:

    db.comments.reply(20, "my_username", "funny! :)")
